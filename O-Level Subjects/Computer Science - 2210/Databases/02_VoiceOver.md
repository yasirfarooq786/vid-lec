# Voice-Over Script: Databases (Cambridge International O Level Computer Science 2210)

**[Visual: Spreadsheet chaos vs neat relational tables]**

**Narrator:**  
Databases organise data so applications can search, update, and keep links **consistent**. Think **tables**, **keys**, and **relationships**.

---

**[Visual: Student table — StudentID primary key highlighted]**

**Narrator:**  
A **primary key** picks one row uniquely. Other tables reference it with **foreign keys**—that is how orders stay tied to real customers.

---

**[Visual: One customer row linked to many order rows]**

**Narrator:**  
**One-to-many** is the bread-and-butter pattern. Draw the crow’s foot or arrows the way your course teaches—consistency matters.

---

**[Visual: SQL window — SELECT … FROM … WHERE Grade = 'A']**

**Narrator:**  
**SELECT** pulls data; **WHERE** filters it. Practise reading criteria aloud: “rows where this condition is true.”

---

**[Visual: Accidental delete icon with warning — orphan rows]**

**Narrator:**  
**Integrity** means you cannot reference missing keys. Exams love “what goes wrong if…” stories—explain the broken link.

---

**[Visual: Checklist — PK/FK, SELECT, INSERT risks, split tables]**

**Narrator:**  
Write five micro-queries this week from past papers. Next: **Security and Ethics**.
